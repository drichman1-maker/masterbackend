**Learned from**: https://github.com/The-Vibe-Company/companion
**Key insights for our stack**:

1. **WebSocket bridging** - Could use for real-time updates in aggregators
2. **Protocol recording** - Log all API calls to JSONL for debugging  
3. **Component playground** - Demo all UI components in one page
4. **Session persistence** - Save subagent state to disk for recovery
5. **NDJSON protocol** - For streaming price updates
6. **Hash routing** - Simple, no-server routing
7. **Zustand state management** - Already using (good choice)
8. **Code organization** - Clean separation under `web/` folder

**Testing patterns**:
- Tests use Vitest, live next to source files
- Pre-commit hooks run typecheck + tests
- Never remove existing tests
